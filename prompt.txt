## What is Runpod Flash?

Runpod Flash is a Python SDK that lets developers write Python functions locally and execute them remotely on Runpod's serverless GPU/CPU infrastructure. The core concept: decorate a function with `@Endpoint(...)`, and Flash handles provisioning GPUs, managing dependencies, serializing data, and returning results — no Docker, no YAML, no infrastructure management.

## Key Concepts

- **Endpoint class**: The single entry point for all Flash functionality. Used as a decorator (`@Endpoint(...)`) for queue-based endpoints, or instantiated (`Endpoint(...)`) for load-balanced HTTP endpoints, custom Docker images, and connecting to existing endpoints.
- **GpuGroup**: Enum for selecting GPU architecture groups (e.g., `GpuGroup.ANY`, `GpuGroup.AMPERE_80`).
- **GpuType**: Enum for selecting specific GPU models (e.g., `GpuType.NVIDIA_GEFORCE_RTX_4090`, `GpuType.NVIDIA_H100_80GB_HBM3`).
- **CpuInstanceType**: Enum for CPU instance types. Can also use string shorthands (e.g., `"cpu5c-4-8"`).
- **PodTemplate**: Override container settings like disk size.
- **NetworkVolume**: Attach persistent storage to endpoints.
- **Dependencies**: Specified in the `@Endpoint(...)` decorator; Flash installs them automatically on the remote worker.
- **Imports inside functions**: All package imports must happen inside the decorated function, not at the top of the file.

## Four Usage Patterns

1. **Queue-based (QB)**: One function per endpoint, decorated with `@Endpoint(...)`. Best for batch processing and fault-tolerant workflows.
2. **Load-balanced (LB)**: Multiple HTTP routes sharing workers via `Endpoint(...)` instance with `.post()`, `.get()`, etc. Best for REST APIs and real-time services.
3. **External image**: Deploy a pre-built Docker image via `Endpoint(name=..., image=...)`.
4. **Existing endpoint**: Connect to an already-deployed endpoint via `Endpoint(id="abc123")`.

## Minimal Example

```python
import asyncio
from runpod_flash import Endpoint, GpuGroup

@Endpoint(
    name="flash-quickstart",
    gpu=GpuGroup.ANY,
    dependencies=["torch", "numpy"],
)
def gpu_compute(data):
    import torch
    import numpy as np

    tensor = torch.tensor(data, device="cuda")
    result = tensor.sum().item()

    return {
        "result": result,
        "device": torch.cuda.get_device_name(0),
    }

async def main():
    result = await gpu_compute([1, 2, 3, 4, 5])
    print(f"Sum: {result['result']}")
    print(f"Computed on: {result['device']}")

if __name__ == "__main__":
    asyncio.run(main())
```

## Endpoint Parameters

| Parameter | Description | Default | Example |
|-----------|-------------|---------|---------|
| `name` | Endpoint name (required unless `id=` used) | — | `"my-worker"` |
| `id` | Connect to existing endpoint by ID | `None` | `"abc123"` |
| `gpu` | GPU type(s) | `GpuGroup.ANY` | `GpuGroup.ADA_24`, `GpuType.NVIDIA_H100_80GB_HBM3` |
| `cpu` | CPU instance type (mutually exclusive with `gpu`) | `None` | `"cpu3c-4-8"`, `CpuInstanceType.CPU5C_4_16` |
| `workers` | Worker scaling: `max` or `(min, max)` | `(0, 3)` | `5`, `(1, 10)` |
| `idle_timeout` | Seconds before scaling down | `60` | `300` |
| `dependencies` | Python packages to install | `None` | `["torch", "numpy"]` |
| `system_dependencies` | System packages to install | `None` | `["ffmpeg"]` |
| `volume` | Network volume for persistent storage | `None` | `NetworkVolume(id="vol_abc")` |
| `datacenter` | Preferred datacenter | `EU_RO_1` | `DataCenter.US_TX_3` |
| `env` | Environment variables | `None` | `{"HF_TOKEN": "xyz"}` |
| `gpu_count` | GPUs per worker | `1` | `2`, `4` |
| `execution_timeout_ms` | Max execution time (ms) | `0` (no limit) | `600000` |
| `flashboot` | Enable Flashboot fast startup | `True` | `False` |
| `image` | Custom Docker image to deploy | `None` | `"vllm/vllm-openai:latest"` |
| `scaler_type` | Scaling strategy | auto | `ServerlessScalerType.QUEUE_DELAY` |
| `scaler_value` | Scaling threshold | `4` | `10` |
| `template` | Pod template overrides | `None` | `PodTemplate(containerDiskInGb=100)` |

## Queue-Based Endpoint Examples

### Basic GPU

```python
from runpod_flash import Endpoint, GpuGroup

@Endpoint(name="my-worker", gpu=GpuGroup.ADA_24, workers=(0, 3))
async def process(data: dict) -> dict:
    return {"result": data}
```

### Worker scaling

```python
# just a max: workers scale from 0 to 5
@Endpoint(name="elastic", gpu=GpuGroup.ANY, workers=5)

# min and max: always keep 2 warm, scale up to 10
@Endpoint(name="always-on", gpu=GpuGroup.ANY, workers=(2, 10))

# default: (0, 3)
@Endpoint(name="default-scaling", gpu=GpuGroup.ANY)
```

### CPU endpoint

```python
@Endpoint(name="data-processor", cpu="cpu5c-4-8", dependencies=["pandas"])
def process(data):
    import pandas as pd
    return pd.DataFrame(data).describe().to_dict()
```

## Load-Balanced Endpoint Examples

```python
from runpod_flash import Endpoint, GpuGroup

api = Endpoint(name="inference-api", gpu=GpuGroup.ADA_24, workers=(1, 5))

@api.post("/predict")
async def predict(data: dict) -> dict:
    import torch
    model = torch.load("/models/model.pt")
    return {"prediction": model.predict(data)}

@api.get("/health")
async def health():
    return {"status": "ok", "gpu": "available"}
```

Key differences from queue-based: direct HTTP routing, lower latency, custom HTTP methods (GET, POST, PUT, DELETE, PATCH), no automatic retries.

Runpod serverless has a **500MB deployment limit**. Which packages to exclude:
- **GPU endpoints** → PyTorch images have torch/torchvision/torchaudio pre-installed
- **CPU endpoints** → Python slim images have NO ML frameworks pre-installed

Build artifacts:
- `.flash/.build/` — Temporary build directory
- `.flash/artifact.tar.gz` — Deployment package
- `.flash/flash_manifest.json` — Service discovery configuration

## Important Rules

1. **Always import packages INSIDE decorated functions**, not at the top of the file.
2. The first run will be slow (~1 min) as the endpoint initializes. Subsequent runs are fast (~1 sec).
3. Remote functions must be awaited (they're async).
4. **500MB deployment limit** for flash build — use `--exclude` for pre-installed packages.
5. GPU and CPU are mutually exclusive. If neither is specified, GPU defaults to `GpuGroup.ANY`.
6. `gpu=` accepts `GpuGroup` or `GpuType` enums only (not strings). `cpu=` accepts string shorthands or `CpuInstanceType` enums.
7. Requires Python 3.10 or higher.
8. Serverless deployments are currently restricted to the **EU-RO-1** datacenter.
9. Endpoints persist until deleted via `flash undeploy` or the Runpod console.
10. Set `workers=(1, N)` to keep workers warm and avoid cold starts.
11. Use `idle_timeout` to balance cost and responsiveness.

## Troubleshooting

**Authentication errors:**
```bash
flash login         # recommended
echo $RUNPOD_API_KEY  # or verify env var
```

**Import errors in remote functions:**
```python
# WRONG
import requests

@Endpoint(name="fetcher", cpu="cpu3c-1-2", dependencies=["requests"])
def fetch_data(url):
    return requests.get(url).json()

# CORRECT
@Endpoint(name="fetcher", cpu="cpu3c-1-2", dependencies=["requests"])
def fetch_data(url):
    import requests
    return requests.get(url).json()
```

**No endpoints found during build:**
- Ensure functions use `@Endpoint(...)` decorator or `Endpoint(...)` instances with registered routes
- Check files aren't excluded by `.gitignore` or `.flashignore`

**Dependency installation failed:**
- Verify the package has pre-built Linux x86_64 wheels on PyPI
- Standard pip recommended over uv pip for best compatibility
